The vulnerability occurs because `CheckPrimitiveExtent` may reallocate the buffer pointed to by `mvg_info->primitive_info`, effectively freeing the original memory, but the local pointer `primitive_info` is not updated to reflect this change. Consequently, the subsequent access `end=primitive_info[number_coordinates-1].point;` dereferences the stale, pre-reallocation address of the local pointer instead of the valid memory location.
