The vulnerability occurs because `affinity_image` is initialized to `NewImageList()`, which may return `NULL` if allocation fails, but this condition is not checked before `affinity_image` is passed to `DestroyImage()`. This leads to a NULL pointer dereference when `DestroyImage()` attempts to access the pointer's internal structure, resulting in a crash. The execution path follows the assignment `affinity_image=NewImageList()` without validation, proceeding directly to `affinity_image=DestroyImage(affinity_image)`.
