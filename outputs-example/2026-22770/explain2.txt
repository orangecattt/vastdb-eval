The vulnerability occurs because `AcquireBilateralTLS` allocates `number_threads+1` pointers but only zeroes `number_threads` entries, creating an uninitialized "valid" pointer at index `number_threads` (used for `spatial_gaussian`) which the deallocation loop attempts to release. Since this specific pointer was never initialized by the memory allocator, passing it to `RelinquishMagickMemory` in `DestroyBilateralTLS` results in the release of an invalid pointer.
