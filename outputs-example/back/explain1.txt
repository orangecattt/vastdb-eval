The vulnerability occurs when exiting a nested DICOM sequence: memory is freed via `info_copy=(DCMInfo *) RelinquishMagickMemory(info_copy)` after `RemoveLastElementFromLinkedList(stack)` retrieves the pointer, but the subsequent use of `info_copy` in the `ThrowDCMException` macro attempts to free this already-freed pointer a second time if an exception is thrown before the pointer is reassigned, creating a use-after-free condition.
