## Root Cause

The vulnerability is CWE-763 (Release of Invalid Pointer or Reference). The issue occurs because `update_all_relocations()` at line 1438 allocates the `all_relocations_root` array BEFORE `dump_relocations()` at line 1440 is called. If `slurp_rela_relocs()` fails inside `dump_relocations()` (line 690 returns false), the function returns early without populating the `r_symbol` fields of the allocated `all_relocations_root` entries. Later, when `process_got_section_contents()` attempts to free each `r_symbol` at line 1673, it tries to free uninitialized/garbage pointers, causing a use-after-free or invalid free.

## Trigger Path

1. `bad.c:1682` `process_object`
   - Entry point that orchestrates ELF file processing. Calls `process_relocs` at line 1741, then later calls `process_got_section_contents` at line 1767.

2. `bad.c:1339` `process_relocs`
   - Processes relocation sections. At line 1438, calls `update_all_relocations` to allocate `all_relocations_root` array based on expected relocation count.

3. `bad.c:670` `dump_relocations`
   - Called at line 1440 from `process_relocs`. At line 690, calls `slurp_rela_relocs` to parse relocation data. If this fails (returns false), the function returns early at line 691 without populating the `r_symbol` fields in the already-allocated `all_relocations_root`.

4. `bad.c:576` `slurp_rela_relocs`
   - Can fail due to allocation failure (lines 593, 604, 623, 634) or invalid data. When it fails, the `rels` output parameter is not set.

5. `bad.c:1509` `process_got_section_contents`
   - Called from `process_object` at line 1767 after `process_relocs` returns. At line 1672-1673, iterates through all `all_relocations_root` entries and calls `free(all_relocations_root[j].r_symbol)`. If `dump_relocations` previously failed, `r_symbol` contains uninitialized garbage, causing an invalid free.
