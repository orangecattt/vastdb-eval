## Root Cause

The vulnerability stems from a **size mismatch** in the `AcquireBilateralTLS` function at line 946. The function allocates an array of `number_threads+1` pointers (line 943), but the subsequent `memset` only zeros out `number_threads` elements (line 946). This leaves `weights[number_threads]` uninitialized with garbage memory. Later, when `DestroyBilateralTLS` iterates from 0 to `number_threads` (inclusive, line 927), it attempts to call `RelinquishMagickMemory` on the uninitialized `weights[number_threads]` pointer (line 929), releasing an invalid pointer/reference—triggering CWE-763.

## Trigger Path

1. `bad.cpp:956` `BilateralBlurImage`
   - Top-level function that calls `AcquireBilateralTLS` and later `DestroyBilateralTLS`

2. `bad.cpp:1006` `AcquireBilateralTLS`
   - Allocates `number_threads+1` pointers but only zeros `number_threads`, leaving `weights[number_threads]` uninitialized (line 946)

3. `bad.cpp:1199` `DestroyBilateralTLS` (called from `BilateralBlurImage`)
   - Iterates `i` from 0 to `number_threads` inclusive (line 927)
   - Attempts to free `weights[number_threads]` which contains garbage/uninitialized memory (line 929)
