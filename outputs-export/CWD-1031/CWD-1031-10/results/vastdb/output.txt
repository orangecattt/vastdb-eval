## Root Cause

The vulnerability is a NULL pointer dereference (CWE-476) in the FIREqualizer filter. The `s->dump_buf` pointer is allocated at line 886 only when `s->dumpfile` is set (inside `if (s->dumpfile)` block at lines 882-887). When the memory allocation fails and returns NULL, the check at line 692 logs a warning but continues execution. If the dump file was successfully opened (making `dump_fp` valid), `dump_fir` is called at line 786. Inside `dump_fir`, at line 646, the code dereferences `s->dump_buf[i]` without checking if `dump_buf` is NULL, causing the NULL pointer dereference.

## Trigger Path

1. `bad.c:802` `config_input`
   - Filter configuration function called when the filter link is configured; calls `generate_kernel` at line 906

2. `bad.c:658` `generate_kernel`
   - Generates the FIR filter kernel; allocates `dump_buf` at line 886 only if `s->dumpfile` is set; at line 692, checks for NULL dump_buf but only logs warning and continues; calls `dump_fir` at line 786 when `dump_fp` is valid

3. `bad.c:599` `dump_fir`
   - Dumps FIR coefficients to file; accesses `s->dump_buf[i]` at line 646 without NULL check, triggering the NULL pointer dereference when `dump_buf` is NULL
