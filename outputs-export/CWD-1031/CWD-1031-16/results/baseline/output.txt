## Root Cause

The vulnerability occurs because `gf_xml_locate_entity()` can return NULL in two scenarios: (1) when an entity is not found in the list, and (2) when the entity name is too short (at line 255-256, it sets `*needs_text = GF_TRUE` and returns NULL). The code at line 504 in `gf_xml_sax_parse_intern` only checks for the first case (`!ent && !needs_text`), but fails to handle the second case where `ent` is NULL and `needs_text` is TRUE. When this happens, execution falls through to line 541 where `ent->value` is dereferenced and passed to `xml_sax_append_string`, triggering the NULL pointer dereference in `strlen(string)` at line 154.

## Trigger Path

1. `bad.c:552` `gf_xml_sax_parse` - Exported function that accepts external XML input and calls `gf_xml_sax_parse_intern` at line 576
