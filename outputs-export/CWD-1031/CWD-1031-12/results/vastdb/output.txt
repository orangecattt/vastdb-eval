## Root Cause

The vulnerability is a NULL Pointer Dereference (CWE-476) in the ALS decoder. The root cause is that `ctx->raw_mantissa[c]` is allocated in `decode_init` (line 774) using `ctx->cur_frame_length` which is **uninitialized at initialization time**. The `cur_frame_length` field is only set in `decode_frame` (lines 1014-1017), which executes **after** `decode_init`. Since `ctx` is zero-initialized memory, `cur_frame_length` starts at 0, causing `av_calloc(0, sizeof(**ctx->raw_mantissa))` to allocate zero bytes and return NULL. When `read_diff_float_data` is subsequently called during decoding, it dereferences these NULL pointers (line 921: `raw_mantissa[c][i] = get_bitsz(gb, nbits[i])`), triggering the NULL pointer dereference.
