## Root Cause

The vulnerability is a NULL Pointer Dereference (CWE-476) in the `output_TEXT` function. The `text_value` field of a TEXT entity is assigned by `bit_read_TV()` in `dwg_decode_TEXT_private()` only when the DWG file version is between R_2000 and R_2007 (line 2233: `if (dat->version >= R_2000 && dat->version < R_2007)`). For versions R_2007 and later, `text_value` is never assigned and remains NULL. When `output_TEXT()` is later called to render the TEXT entity, it directly uses `text->text_value` at line 1972 without checking for NULL, causing the NULL pointer dereference when `htmlescape()` is called.
