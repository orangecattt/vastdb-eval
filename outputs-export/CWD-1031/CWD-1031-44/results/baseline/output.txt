## Root Cause

The vulnerability is a NULL pointer dereference in CWE-476. The `text_value` field of `Dwg_Entity_TEXT` is set only when the DWG file version is between R_2000 and R_2006 (exclusive of R_2007). In `dwg_decode_TEXT_private` (line 2233-2234), the assignment `_obj->text_value = bit_read_TV(dat)` only executes for this version range. For R_2007 and later versions, `text_value` is never assigned and remains NULL. Later, in `output_TEXT` (line 1972), the function `htmlescape(text->text_value, ...)` is called without checking if `text->text_value` is NULL, causing a NULL pointer dereference when the function tries to read from the NULL pointer.

## Trigger Path

1. `bad.c:3585` `main`
   - Entry point; calls `dwg_read_file` to parse the DWG file, then calls `output_SVG`

2. `bad.c:3682` `dwg_read_file`
   - Parses the DWG file and populates the `Dwg_Data` structure, including calling `dwg_decode_TEXT` for TEXT entities

3. `bad.c:3684` `output_SVG`
   - Calls `output_BLOCK_HEADER` for model space which iterates through entities

4. `bad.c:2100` (approximate) `output_BLOCK_HEADER`
   - Iterates through entities using `get_first_owned_entity`/`get_next_owned_entity`, calling `output_object` for each

5. `bad.c:2006` `output_object`
   - Switches on object type and calls `output_text` for DWG_TYPE_TEXT

6. `bad.c:1957` `output_TEXT`
   - Calls `htmlescape(text->text_value, ...)` at line 1972 without NULL check
   - `text->text_value` is NULL when the DWG version is R_2007 or later (or other versions where text_value was never set), causing the NULL pointer dereference
