## Root Cause

The vulnerability is a Use-After-Free (CWE-416) in the RealMedia demuxer. The `mime` string parsed from the file header is passed through `rm_read_multi` to `ff_rm_read_mdpr_codecdata`. When `mime` equals "logical-fileinfo", the function at line 640 calls `ff_free_stream(s, st)` which frees the stream object. However, the function returns success (0) instead of an error, and the caller (`rm_read_multi`) is unaware that the stream has been freed. The original `st` pointer in `rm_read_header` now points to freed memory, and any subsequent use of this dangling pointer triggers the use-after-free.

## Trigger Path

1. `bad.c:757` `rm_read_header`
   - Entry point that parses the RM file header. At line 831, reads the MIME type from the file. At line 843, calls `rm_read_multi` passing the `mime` string.

2. `bad.c:715` `rm_read_multi`
   - At lines 748-749, calls `ff_rm_read_mdpr_codecdata(s, s->pb, st2, st2->priv_data, size2, mime)`, passing the `mime` string to the next function.

3. `bad.c:607` `ff_rm_read_mdpr_codecdata`
   - At line 638, checks `if(mime && !strcmp(mime, "logical-fileinfo"))`. When true, at line 640 calls `ff_free_stream(s, st)` which frees the stream but returns success (0) instead of an error. The caller retains a pointer to the freed memory, causing use-after-free when subsequently accessed.
