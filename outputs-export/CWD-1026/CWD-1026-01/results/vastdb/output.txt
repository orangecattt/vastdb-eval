## Root Cause

The vulnerability is a **Use-After-Free (CWE-416)** in the FFmpeg RM demuxer code. In `ff_rm_read_mdpr_codecdata`, when the `mime` parameter equals "logical-fileinfo", the function calls `ff_free_stream(s, st)` at line 640, freeing the `AVStream` object. However, the calling function `rm_read_multi` does not expect the stream to be freed at this point and may continue to use the freed pointer, leading to use-after-free. The sink check `if(mime && !strcmp(mime, "logical-fileinfo"))` at line 638 triggers the stream deallocation, but the caller is unaware of this side effect.
