## Root Cause

The vulnerability is in the `streamGetEdgeID` function in `bad.c`. A `streamIterator` struct is created on the stack at line 421, and `streamIteratorStart()` is called at line 423 (the source) which initializes the iterator and potentially allocates heap memory through the internal `raxIterator`. When the function returns at line 430 (the sink), it never calls `streamIteratorStop()` to free this memory, resulting in a memory leak (CWE-401).

## Trigger Path

1. `bad.c:419` `streamGetEdgeID`
   - Top-level function that retrieves edge stream IDs; allocates a `streamIterator` on the stack and returns without cleanup.

The VAST DB queries confirmed:
- Function `streamGetEdgeID(stream *, int, int, streamID *)` exists (uid 531)
- The call to `streamIteratorStart` at line 423 is present (`hl.call` at location 423:5)
- The return statement at line 430 is present (`hl.return` at location 430:5)
- No `streamIteratorStop` call exists within the function, confirming the missing release of memory.
