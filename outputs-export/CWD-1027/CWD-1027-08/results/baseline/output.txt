## Root Cause

In `ff_frame_thread_encoder_init` (bad.c:1426), multiple `AVCodecContext` objects are allocated in a loop at line 1506 using `avcodec_alloc_context3()`. When any subsequent operation fails (e.g., `av_opt_copy`, `avcodec_open2`, or `pthread_create`), the code jumps to the `fail:` label at line 1536, which calls `ff_frame_thread_encoder_free(avctx)` but does NOT free the `thread_avctx` contexts allocated in the loop. The function then returns -1 at line 1540, causing a memory leak (CWE-401).

## Trigger Path

The code in this directory is self-contained and does not include a full stack trace with external callers. Based on the provided information:

1. `bad.c:1426` `ff_frame_thread_encoder_init`
   - This is the top-level function containing the vulnerable allocation and sink. No caller is defined in this file; it would be called from external FFmpeg code (e.g., `avcodec_open2`). The vulnerability triggers when any operation after line 1506 fails and the function returns -1.

The memory allocated at line 1506 (`AVCodecContext *thread_avctx = avcodec_alloc_context3(avctx->codec);`) is never freed on the failure path, as the `fail:` cleanup does not release these per-thread contexts.
