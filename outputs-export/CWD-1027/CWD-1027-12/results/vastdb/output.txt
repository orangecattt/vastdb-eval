## Root Cause

The vulnerability is a **CWE-401 Missing Release of Memory after Effective Lifetime**. In the `glutAddMenuEntry` function (bad.c:269-286), memory is allocated via `calloc` at line 273 for `SFG_MenuEntry *menuEntry`. When the condition at line 276 evaluates to true (i.e., `fgState.ActiveMenus` is non-zero), `fgError` is called at line 277. The `fgError` function (bad.c:236-267) unconditionally calls `exit(1)` at line 265, terminating the program without freeing the previously allocated `menuEntry`. Since `exit()` bypasses normal cleanup and there's no cleanup handler for this specific allocation, the memory is never released before program termination.

## Trigger Path

1. `bad.c:269` `glutAddMenuEntry(const char *label, int value)`
   - External caller invokes this function with a menu label and value.
   - Line 273 allocates `SFG_MenuEntry` via `calloc` before any checks.

2. `bad.c:276` (if condition)
   - Checks `if (fgState.ActiveMenus)` - if true, control flows to error path.

3. `bad.c:277` `fgError("Menu manipulation not allowed while menus in use.")`
   - Called when menus are in use; this is the sink location.

4. `bad.c:265` `exit(1)` (inside `fgError`)
   - Called from `fgError` function (bad.c:236), unconditionally terminates program.
   - The allocated `menuEntry` at line 273 is never freed.
