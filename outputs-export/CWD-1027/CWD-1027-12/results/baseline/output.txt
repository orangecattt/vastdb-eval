## Root Cause

The vulnerability is in `glutAddMenuEntry` (bad.c:273). A `SFG_MenuEntry` structure is allocated via `calloc()`. When `fgState.ActiveMenus` is non-zero (bad.c:276), the code calls `fgError()` (bad.c:277), which prints an error and exits the program (bad.c:265). The allocated memory is never freed before the program terminates, causing a memory leak (CWE-401).

## Trigger Path

1. `bad.c:269` `glutAddMenuEntry`
   - External/top-level API entry point. Receives menu label and value from caller.
   - Allocates `menuEntry` via `calloc` at line 273
   - Checks `fgState.ActiveMenus` at line 276; if true, calls `fgError()` which exits without freeing the allocated memory

Note: This is a library API function. The test case contains only the library implementation, not the calling code. The function `glutAddMenuEntry` itself is the external entry point that would be invoked by user code to trigger this vulnerability.
