## Root Cause

The vulnerability is a CWE-787 Out-of-bounds Write in `evp_cipher_get_asn1_aead_params`. The function makes two sequential calls to `ossl_asn1_type_get_octetstring_int`: first with `data=NULL` to retrieve the ASN1 data length, then using that returned length as the `max_len` parameter for the second call. The problem is the second call's `data` buffer (`iv`) is sized at `EVP_MAX_IV_LENGTH` (typically 16 bytes for AES), but the length returned from the first call can exceed this buffer size. This causes the `memcpy` in `asn1_type_get_int_oct` (line 199) to write beyond the buffer boundaries.
