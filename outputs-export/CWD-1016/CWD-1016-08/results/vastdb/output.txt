## Root Cause

The vulnerability is a CWE-787 Out-of-bounds Write in `slice_get_derivative` at line 505 where `gauss[GINDX(filtersize, g)]` is accessed. The `GINDX` macro is defined as `(i) - ((s) >> 2)` which equals `g - filtersize/4`. When `filtersize` is small (such as 5 or smaller), this calculation produces negative indices. Specifically, with `filtersize = 5`, `GINDX(5, 0) = 0 - 1 = -1`, causing an out-of-bounds write to negative memory offset. The source at line 385 allocates `s->gauss[i]` arrays of size `filtersize`, but does not validate that `GINDX(filtersize, g)` produces valid (non-negative) indices.

7. `bad.c:505` `slice_get_derivative` (sink)
   - The `gauss[GINDX(filtersize, g)]` write causes out-of-bounds when `filtersize < 4` (e.g., `filtersize=5` produces `GINDX(5,0)=-1`)
