## Root Cause

The vulnerability is a CWE-787 Out-of-bounds Write caused by a stride mismatch between the pre-computed buffer stride and the actual frame linesize. In `ff_vmafmotion_init`, `s->stride` is computed as `FFALIGN(w * sizeof(uint16_t), 32)` based on the input width `w` (line 405), and `s->temp_data` is allocated with size `s->stride * h` (line 410). Later in `ff_vmafmotion_process`, the `convolution_y` function is called with `s->stride` as the destination stride but uses `ref->linesize[0]` from the actual input frame as the source stride (line 497-498). When `ref->linesize[0]` differs from the pre-computed `s->stride` (which can happen due to different padding/alignment in video frames), the convolution operation can write beyond the allocated buffer bounds, causing an out-of-bounds write.
