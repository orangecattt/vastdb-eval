## Root Cause

The vulnerability is a classic buffer overflow in the `append_command` function at line 351 where `STRCAT(IObuff, ": ");` is called without checking if `IObuff` has sufficient space to accommodate the appended string. The global buffer `IObuff` is defined with size `IOSIZE` (1025 bytes) at line 97. Before calling `append_command`, the buffer may already contain data from prior operations (e.g., error messages copied via `STRCPY(IObuff, _(e_not_an_editor_command))` at line 663). The function unconditionally concatenates ": " using `strcat` without validating available space, allowing an attacker-controlled input to fill the buffer and cause an out-of-bounds write when additional content is appended.
