## Root Cause

The vulnerability is a CWE-787 Out-of-bounds Write in `deflate16()`. The function receives a `dst` pointer and writes `dst[x]` in a loop for `x` from 0 to `width-1`. Since `dst` is cast to `uint16_t*` (line 334), each write is 2 bytes, totaling `2*width` bytes. However, in `filter_slice` (line 388), `dst` is passed as `dst + 1*bpc`, where `bpc` is bytes per component (typically 2 for 16-bit video). The caller allocates only `width*2` bytes for the row, so when `deflate16` writes starting from offset `bpc` (2 bytes), it overflows by 2 bytes beyond the buffer end. The sink at line 343 (`dst[x] = ...`) directly writes to this overflowed memory.
