## Root Cause

The vulnerability is an **out-of-bounds write** caused by an improper bounds check. The `eval_result` array has size `MAX_REGSUB_NESTING` (4), so valid indices are 0-3. In `vim_regsub_both()` at line 393, the check `if (nesting == MAX_REGSUB_NESTING)` only triggers when `nesting` equals 4, but not when it exceeds 4. When `nesting` is 3, `nested` is set to 3, then `nesting` is incremented to 4 before recursive evaluation (line 445). After the recursive call returns and `nesting` is decremented back to 3, the `nested` value can still exceed the array bounds, causing `eval_result[nested]` at line 418 to access memory beyond the allocated array, leading to an out-of-bounds write via `STRCPY(dest, eval_result[nested])`.
